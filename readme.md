# AutoRia Clone API (Node.js & MongoDB)

Це сучасна масштабована платформа для продажу автомобілів, розроблена на базі Node.js з використанням архітектури, що готова до розгортання в **AWS**. Проєкт спроектований так, щоб витримувати високі навантаження та легко розширюватися для роботи з автосалонами.

---

##  Архітектурні рішення
* **База даних:** **MongoDB** — забезпечує гнучкість збереження даних про авто з різними характеристиками.
* **Контейнеризація:** Повна підтримка **Docker** для легкого деплою в AWS ECS.
* **Система дозволів (RBAC):** Гнучка модель прав доступу, що враховує майбутні ролі (механіки, сейли автосалонів).
* **Currency Engine:** Автоматична конвертація цін на основі актуального курсу ПриватБанку (оновлюється раз на добу, реалізовано з використанням Mock-даних для стабільності тестування).

---

##  Технологічний стек
* **Runtime:** Node.js (v18+)
* **Language:** TypeScript
* **Database:** MongoDB (Mongoose)
* **Documentation:** Swagger / OpenAPI 3.0
* **Validation:** Joi
* **Task Scheduling:** Node-cron (для оновлення валют)

---

##  Робота з медіа (AWS S3)
Проєкт використовує хмарне сховище **Amazon S3** для збереження фотографій автомобілів.
* Реалізовано безпечне завантаження зображень.
* Згенеровані посилання зберігаються в БД та використовуються для відображення оголошень.
* Налаштовано підтримку AWS SDK для Node.js.

### Варіанти запуску проєкту

Проєкт підтримує два режими роботи:

1.  **Full Docker Stack** (рекомендовано для деплою/демонстрації):
    Усі сервіси (App, DB, Nginx) піднімаються однією командою:
    ```bash
    docker-compose up --build
    ```
    Доступ до сервісів:

Backend API: http://localhost:5000 (mapping 5000:7000)

Swagger UI: http://localhost:5000/api-docs

Frontend (Nginx): http://localhost:80

MongoDB: доступна локально на порту 1234

2.  **Hybrid Dev Mode** (найзручніший для розробки):
    Дозволяє використовувати `hot reload` для Node.js.
    * Запустіть тільки базу даних: `docker-compose up -d db`
    * Запустіть сервер локально: `npm run start:dev`

docker-compose up --build

### Налаштування середовища
Створіть файл `.env` у корені:
приклад фалу .env: .env.example - додано до проекту

```env
# Порт 7000 необхідний для коректної роботи Docker (mapping 5000:7000)
PORT=7000

# Для локального запуску (Hybrid Mode) використовуйте порт 1234
MONGO_URI=mongodb://user:user@localhost:1234/nodejs-auto-ria-db?authSource=admin

# Секретний ключ для авторизації
JWT_SECRET=your_super_secret_key

# Налаштування AWS S3
AWS_ACCESS_KEY=your_key
AWS_SECRET_KEY=your_secret
AWS_S3_BUCKET=your_bucket_name
AWS_S3_REGION=your_region
```
