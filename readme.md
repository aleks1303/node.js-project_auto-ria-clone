# AutoRia Clone API (Node.js & MongoDB)

Це сучасна масштабована платформа для продажу автомобілів, розроблена на базі Node.js з використанням архітектури, що готова до розгортання в **AWS**. Проєкт спроектований так, щоб витримувати високі навантаження та легко розширюватися для роботи з автосалонами.

---

##  Архітектурні рішення
* **База даних:** **MongoDB** — забезпечує гнучкість збереження даних про авто з різними характеристиками.
* **Контейнеризація:** Повна підтримка **Docker** для легкого деплою в AWS ECS.
* **Система дозволів (RBAC):** Гнучка модель прав доступу, що враховує майбутні ролі (механіки, сейли автосалонів).
* **Currency Engine:** Автоматична конвертація цін на основі актуального курсу ПриватБанку (оновлюється раз на добу, реалізовано з використанням Mock-даних для стабільності тестування).

---

##  Технологічний стек
* **Runtime:** Node.js (v18+)
* **Language:** TypeScript
* **Database:** MongoDB (Mongoose)
* **Documentation:** Swagger / OpenAPI 3.0
* **Validation:** Joi
* **Task Scheduling:** Node-cron (для оновлення валют)

---

###  Монетизація та типи акаунтів
* **Base Account (Базовий):**
    * Обмеження на створення оголошень — лише **1 активне авто**.
    * Стандартний функціонал для звичайних користувачів.
* **Premium Account (Преміум):**
    * **Необмежена кількість** оголошень для великих продавців.
    * **Аналітика переглядів:** доступ до статистики за день, тиждень та місяць.
    * **Ринковий аналіз:** автоматичний розрахунок середньої ціни по Україні та регіону на аналогічні авто.

---

---

### ️ Розумна модерація оголошень
* **Фільтрація контенту:** Система автоматично перевіряє описи оголошень на наявність нецензурної лексики.
* **Система спроб:** Користувачу надається **3 спроби** на самостійне виправлення тексту, якщо знайдено порушення.
* **Автоматичне блокування:** Якщо після 3-ї спроби текст не пройшов валідацію:
    * Оголошення автоматично отримує статус `inactive`.
    * Оголошення блокується для публічного перегляду.
    * Система надсилає email-сповіщення **менеджеру** для ручної перевірки та модерації.

##  Робота з медіа (AWS S3)
Проєкт використовує хмарне сховище **Amazon S3** для збереження фотографій автомобілів.
* Реалізовано безпечне завантаження зображень.
* Згенеровані посилання зберігаються в БД та використовуються для відображення оголошень.
* Налаштовано підтримку AWS SDK для Node.js.

---
### Варіанти запуску проєкту

Проєкт підтримує два режими роботи:

1.  **Full Docker Stack** (рекомендовано для деплою/демонстрації):
    Усі сервіси (App, DB, Nginx) піднімаються однією командою:
    ```bash
    docker-compose up --build
    ```
    Доступ до сервісів:

Backend API: http://localhost:5000 (mapping 5000:7000)

Swagger UI: http://localhost:5000/api-docs

Frontend (Nginx): http://localhost:80

MongoDB: доступна локально на порту 1234

2.  **Hybrid Dev Mode** (найзручніший для розробки):
    Дозволяє використовувати `hot reload` для Node.js.
    * Запустіть тільки базу даних: `docker-compose up -d db`
    * Запустіть сервер локально: `npm run start:dev`
 
---

### Особливості логіки та безпеки

1. **Реєстрація Адміністратора:** Використовується секретний ключ, який ви маєте вказати у файлі `.env` (змінна `ADMIN_KEY`). Цей же ключ потрібно передати в тілі запиту при створенні адміна.
2. **Ієрархія користувачів:** Адміністратор створює Менеджерів через спеціальний ендпоінт. Після створення менеджер має пройти процедуру `signIn` для активації доступу.
3. **Автоматизація (Seeding):** Списки автомобілів (марки та моделі) ініціалізуються автоматично при першому запуску додатка.
4. **Верифікація через Email:**
    - Додаток підтримує реальну відправку листів через Gmail SMTP (потрібно вказати `EMAIL_USER` та `EMAIL_PASS` у `.env`).
    - **Для швидкої перевірки:** Посилання для верифікації або зміни пароля автоматично дублюється в **логи Docker-контейнера** (шукайте повідомлення `VERIFICATION LINK FOR...` у терміналі). Це дозволяє активувати акаунт без налаштування SMTP.
5. **Захист паролів:** Впроваджено механізм перевірки історії. Користувач не може встановити пароль, який вже використовувався ним протягом останніх **180 днів**.

### Налаштування середовища
Створіть файл `.env` у корені:
приклад фалу .env: .env.example - додано до проекту

```env
# Порт 7000 необхідний для коректної роботи Docker (mapping 5000:7000)
PORT=7000

# Для локального запуску (Hybrid Mode) використовуйте порт 1234
MONGO_URI=mongodb://user:user@localhost:1234/nodejs-auto-ria-db?authSource=admin

# Секретний ключ для cтворення адміністратора
ADMIN_KEY=your_super_secret_key


# Налаштування AWS S3
AWS_ACCESS_KEY=your_key
AWS_SECRET_KEY=your_secret
AWS_S3_BUCKET=your_bucket_name
AWS_S3_REGION=your_region
```

##  Корисні команди для розробки

Якщо під час розробки або розгортання виникають питання з контейнерами, використовуйте наступні сценарії:

* **Повне перезавантаження проєкту (Hard Reset):**
  Видаляє старі контейнери, очищує об'єми (volumes) та перезбирає образи з нуля.
    ```bash
    docker-compose down -v
    docker-compose up --build
    ```

* **Очищення бази даних:**
  Якщо потрібно повністю зачистити дані MongoDB (наприклад, для тестування ініціалізації з нуля):
    ```bash
    docker-compose down -v
    rm -rf mongo_db
    ```

* **Запуск у фоновому режимі (Detached mode):**
    ```bash
    docker-compose up -d
    ```

* **Виправлення проблем із залежностями (Build без кешу):**
    ```bash
    docker-compose build --no-cache
    ```

* **Робота в гібридному режимі (Тільки БД):**
  Запуск лише контейнера з базою даних для локального дебагу через `npm run start:dev`:
    ```bash
    docker-compose up -d db
    ```
